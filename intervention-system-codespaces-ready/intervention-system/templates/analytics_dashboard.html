Demonstrating Ontological Query Capabilities

These questions cannot be answered with traditional intervention notes, but are answerable automatically through ontological structure:

"Which participants received goal-setting techniques?"
Query: Find all encounters containing BCT instances typed as bcio:Goal_Setting_Behaviour
"Do capability barriers respond differently than motivation barriers?"
Query: Group barrier instances by type, calculate average change for each. Automatic classification through COM-B framework.
"Which BCTs target the mechanisms needed for identified barriers?"
Query: Link barriers → mechanisms they require → BCTs that target those mechanisms. Enables automatic intervention matching.
"Compare Protocol v1.0 vs v2.0 effectiveness"
Query: Group encounters by protocol version instance, compare outcomes. Temporal versioning enables valid comparison across protocol evolution.
[Key Point] The Value Proposition
Traditional intervention notes require manual chart review to answer these questions. With ontological structure, answers are automatic, systematic, and scalable. Every encounter automatically generates analyzable evidence about which specific techniques, targeting which mechanisms, reduce barriers for which populations.
